Kollaborasjon i GITHUB

Hvorfor skal man laste ned programvare for å sammarbeide på prosjekter? Kan man ikke bare sende over filene eller endringene på mail?

Kollaborasjon

GIT, Github, Liveshare eller lignende programmer gjør det mye lettere å sammarbeide på prosjekter, nettsider og programmer.

Man slipper å sende over filer, og det tillater flere å jobbe på samme kodebase samtidig.

Historie

Ikke bare kan man sammarbeide samtidig. Men alle endringer utført av personer blir stemlet med både hvilke endringer som er gjort, filer som fjernet, osv. Dato, og hvem som har gjort det.

Gjorde en feil? Sletta hele kodebasen?

Siden alle endringer er loggførte, vil det være enkelt å rett og slett rollbacke - eller gå tilbake i tid og gjenskape prosjektet slik det var for x-endringer siden.

Videre utviling

Når man bruker disse verktøyene kan man legge til tillegsinformasjon som kommentarer, tittel, og beskrivelser

Dette gjør at om en ekstern utvikler eller en ny utvikler ser igjennom endringene, vil han fort og enkelt skjønne hva som er blitt endret i de. Det gjør kodebasen tryggere og sikrere med tanke på om f.eks den ansvarlige slutter.

Open-Source

Med bruk av for eksempel GITHUB, kan man hoste og delta i open-source prosjekter. For eksempel Se Linux sin GITHUB repo (github.com). Der kan alle som ønsker, delta i prosjeket.

På den måten kan store prosjekter bli laget av store mengder med utviklere.

Hvorfor bruke GIT

Fordeler med GIT

  • Open-Source Samarbeid
  • Flere kan jobbe på samme prosjekt samtidig
  • Versjon kontroll sikrer at feil ikke ødelegger kodebasen.
  • Alle endringer er loggført med dato, person, endring.
  • Tryggere da man slipper å sende kode over kommunikasjons nettverk
  • Man kan teste på egne test-brancher uten å påvirke hoved branchen.
